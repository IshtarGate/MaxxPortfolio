Git and Version Control




Version control is a must have in the software development world. Being able to synchronize work across teams and co-workers, makes small projects easy, and big impossible projects, possible.

I have developed proficiency with:

  • Git pull, branch, merge, and push - for standard access of repositories and committing changes to the develop branch
  • Git reset/revert - to erase mistakes, such as bad commits, or create new commits that reverse the previous bad commit
  • Git log and gitk - to examine commit history, including SHA1 keys, or show commit history interactively
  • Git Flow -
    • Git flow init - to initialize git flow in a directory for the first time
    • Git flow feature start my_feature_name - to create a new feature with the specified name
